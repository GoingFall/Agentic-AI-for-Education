# Gemini 战略分析摘录：功能设想与工程实现

> 来源：*Foundations of Agentic AI for Science and Engineering Education*（Gemini 对 HackaStone 2026 的战略分析）。以下为对**功能设想**与**工程实现**有直接参考价值的中文整理。

---

## 一、功能设想

### 1. 从「把事情做得更好」到「做更好的事」

- **Doing Things Better**：自动化传统教学与行政（如生成教案、总结阅读、做 PPT）。有价值，但不足以应对「颠覆、混乱、动荡」的全球文明。
- **Doing Better Things**（更值得追求）：
  - 支持**学生主导的协作设计**；
  - **个性化学习路径**达到前所未有的粒度；
  - 重点培养**素养/倾向**：韧性、坚持、创造力。
- 在理工教育中的体现：不做「替学生解方程」，而是做**虚拟实验伙伴**——推动归纳推理、实时发现程序性缺口、管理**闭环发现过程**。

### 2. STEM 中 Agent 的四种认知环节

| 环节 | 含义 |
|------|------|
| **观察** | 通过数据流感知环境，如学生在 PhET-iO 物理仿真中的实时操作。 |
| **推理** | 用 LLM 或物理信息神经网络（PINNs）从观察得出结论，例如发现学生把力臂长度与总距离混淆。 |
| **规划** | 制定多步策略纠偏，如先做单位换算练习再回到主仿真。 |
| **行动** | 执行计划：数字侧（发送定向提示）或具身侧（调整机械臂轨迹）。 |

- 与被动助手的区别：Agent **主动监测**表现，在**「理想挑战区间」**介入——帮助既不过低（挫败）也不过高（失去兴趣）。

### 3. 获奖项目应具备的特质

- 超越**「聊天机器人」**，提供**「数字队友」**：能在高教生态中**观察、决策、行动**。
- 评审维度：创意、技术深度、可用性、案例连贯性、**团队协作效能**。
- 可参考的获奖方向（近年 Agent 类赛事）：
  - 语义化多社交网络、全栈上下文；
  - **跨请求的持久化、上下文感知会话**；
  - **多智能体学业规划**（如 OnCourse）；
  - **个性化辅导多智能体**（如 PolyTutor）；
  - **目标驱动的个性化学习需求评估**（如 Agentic Neurodiversity Assessment）。

### 4. 伦理与教学对齐（Ethical Presence）

- **促进高阶思维**：搭建脚手架、引导过程，而非只给最终答案。
- **可解释与透明**：学生应知道在与 Agent 交互，并理解 Agent 的推理依据。
- **支持学生声音**：作为引导者鼓励学生自己的写作/解题风格，而非强推统一「AI 风格」。

### 5. 功能聚焦：「杀手级钩子」与人设

- 找准**一个关键痛点**，例如：结构力学课中**管理成千上万份学生推导**的难度。
- 明确 **Agent 人设**：面向研究生的是「技术研究助理」，还是面向本科的「自适应实验教练」？人设决定能力边界与交互设计。

## 二、工程实现

### 1. 框架选型（STEM 项目参考）

| 框架 | 开发者画像 | 架构特点 | 教育场景优势 |
|------|------------|----------|--------------|
| **LangGraph** | 进阶 | 图结构（有向环图 DCG） | 复杂分支、循环、「时间旅行」调试；适合**假设检验—实验—反思—重规划**的非线性科学探究。 |
| **CrewAI** | 入门～中级 | 角色协作 | 将 Agent 角色映射到人类团队（如研究员/写手），快速原型。 |
| **AutoGen** | 中级 | 对话式 | 多智能体协作与人在环（human-in-the-loop）支持强。 |
| **LlamaIndex** | 中级 | RAG 优先 | 数据与知识密集型 Agent、检索 grounding 表现好。 |
| **Agno** | 专业 | 统一/模型无关 | 高性能、生产级 Python 框架。 |
| **n8n** | 低代码/无代码 | 可视化工作流 | 400+ 预制集成，降低 Agent 搭建门槛。 |

- **建议**：需要**复杂迭代与闭环**（如科学探究中的「推理—行动」循环）时，优先考虑 **LangGraph**；若先做**角色分工明确**的协作原型，可用 **CrewAI**。

### 2. 反思模式（Reflection Pattern）——保证 STEM 正确性

- **问题**：LLM 可能产生**看似合理但错误**的科学/数学表述（幻觉），理工教育不能接受。
- **做法**：增加**反思节点（Reflection Node）**，在答案呈现给学生**之前**，用**已知真值或第二套推理模型**校验输出。
- 数学求解器示例中，反思节点可检查：
  1. **指数与阶乘**：高优先级运算符是否先处理；
  2. **工具调用**：每一步是否调用了正确的计算工具（如乘 vs 加）；
  3. **逻辑一致**：推导是否缺步或重复。
- 任一检查失败则进入**自校正循环**，避免错误传递到学生界面。

### 3.  grounding：科学与数学能力

- Agent 在理工教育中的价值，直接取决于**是否被正确 grounding**——即能否调用**权威外部知识与计算引擎**。

**Wolfram Alpha**

- 与 LLM 的互补：LLM 做概率性文本，Wolfram 做**确定性符号计算**。
- 适用场景：精确数学计算（微分方程、积分、矩阵）、科学数据查询（物理常数、材料属性、天文数据）、**单位换算与量纲一致**（如保证学生答案与输入单位一致）。
- **工程建议**：用 LangChain 等将 Wolfram Alpha API 封装为 Agent 的 **Tool**，可快速建立纯 LLM 项目难以达到的**可靠性护城河**。

**PhET-iO（虚拟实验）**

- 通过 **wrapper** 与 PhET-iO 元素通信（如 `phetioID` 字符串）。
- 可实现：**监测状态**（如读取气体温度）、**支架式发现**（若学生乱点则禁用部分控件并先要求预测）、**采集证据**（如自动截取电路配置图用于实验报告）。

**NASA 开放 API**

- 适合航天、环境工程等：**DONKI**（空间天气）、**NeoWs**（近地天体）、**Mars Weather**（火星车气象）、**APOD**（每日天文图）、**code.nasa.gov**（开源代码与 AI/仿真项目）。
- 将真实任务数据接入 Agent 工作流，实现**从理论学习到数据驱动探索**的升级。

### 4. 高等教育中的 RAG：从向量到图

- 高教内容**密集、领域专深**，仅靠**向量相似度检索**的 RAG 难以表达**层级与关系**。
- **GraphRAG**：将文档组织成**实体—关系**知识图。例如「加速度」不仅是一个词，而是与「力」（牛顿第二定律）、「时间」（运动学）相关联的概念；适合**主题式提问**（如「这三个实验如何体现能量守恒？」）和**跨学期课程综合**。
- **对比简要**：
  - **向量 RAG**：速度快、成本低、简单问答够用；多跳推理弱、上下文易膨胀。
  - **图 RAG**：多跳综合、概念层级、准确度高；资源消耗约 10–20 倍、搭建复杂。
  - **混合 RAG**：结合两者，需设计**路由逻辑**（何时走向量、何时走图）。

### 5. 自动批改与 RAG 的可靠性

- 研究显示：基于 RAG 的 Agent 可达到与人工评分**94%–99% 一致**的等级与形成性评语。
- **关键**：采用**「Instruction-Level Policies」**，将 Agent 输出**严格限定在给定知识库/评分标准内**（即 **grounding to the rubric**），按课程具体学习目标评价，而非泛化互联网信息。

### 6. 数据与伦理治理

- 学生已大量将 AI 纳入学习流程，学校易形成**无法管控的「影子系统」**。
- 建议：**有状态 Agent 工作流**、符合 FERPA/GDPR 等；**代码沙箱执行**；**严格访问控制**——每位学生仅能访问自己的数据。

### 7. 一周落地路线图（HackaStone 级项目）

| 天数 | 主题 | 要点 |
|------|------|------|
| **Day 1** | 范围与人设 | 定义「杀手级钩子」、关键痛点；确定 Agent 人设（如技术研究助理 vs 自适应实验教练）。 |
| **Day 2** | 架构与数据 | 选定编排框架（角色协作用 CrewAI，复杂循环用 LangGraph）；搭建 RAG 管道，摄入课程大纲、PhET-iO 文档、教材 PDF。 |
| **Day 3** | Grounding 与工具 | 接入 Wolfram Alpha LLM API、PhET-iO wrapper；实现**反思模式**；支持「输入解释」「分步解答」等模块。 |
| **Day 4** | 主动性与状态 | 不做被动等提示：用**监测循环**检测学生「疲劳/停滞」并触发提示；实现**跨会话状态持久化**。 |
| **Day 5** | 测试与评估 | 用 Shapiro-Wilk 或前后表现对比生成数据；用 CoI 等量表评估社会存在、伦理存在是否到位。 |
| **Day 6** | 视频与文档 | 录制 8 分钟提交视频：展示**推理过程**（内部日志、Wolfram 调用、自校正步骤）；突出**「为何重要」**（对学生韧性、教师负担的实际影响）。 |
| **Day 7** | 路演与提交 | 定稿 15 页 pitch；明确商业模式（如 freemium vs 机构授权）、数据/护城河优势；核对 HackaStone 报名与提交要求。 |

## 三、一句话目标（原文）

> Agentic AI 在理工教育中的目标，是弥合**「知—行」鸿沟**：让成熟的计算能力与学生、科学家的实际需求之间的空隙越来越小；通过**可靠、情境感知、主动**的支持，赋能下一代以更强的韧性与创造力应对世界级复杂挑战。
