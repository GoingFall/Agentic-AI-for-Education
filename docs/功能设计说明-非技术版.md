# 本项目功能设计说明

---


我们做的是一个 **《信号与系统》课程的小助手** ：学生像聊天一样提问，助手**按教材内容**回答并告诉你是哪一章哪一页，还能帮你**理清整本书的知识结构**、**推荐下一步该学什么、练什么**。

---

## 这个助手能做哪几件事？

| 能做的事 | 用大白话说 |
|----------|------------|
| **1. 答疑** | 你问概念、解题思路、实验步骤等，助手去「翻」我们灌进去的教材和讲义，用那里的内容回答你，并**标明出处**（例如「见第 3 章第 5 页」），而不是随便编。 |
| **2. 推荐下一步** | 学完某一块内容后，助手可以根据「先学什么、后学什么」的关系，建议你**下一步学哪一章**或**做哪一份练习**，并简单说理由。 |
| **3. 知识框架** | 你（或我们）**输入一本教材的电子版（如 PDF）**，系统能整理出这本书的**知识框架**：章节目录、有哪些知识点、它们之间谁先谁后，方便你把握全书结构。首版先对《信号与系统》这一本跑通；以后可以支持你上传自己的教材，也生成框架。 |
| **4. 预留：薄弱点（以后做）** | 设计里已经留好位置，以后可以分析你「哪里常错、哪里不熟」，并针对薄弱点推荐复习和练习。首版先不做这块逻辑，只把展示和数据结构留好。 |

---

## 学生用起来是什么样？（流程）

```
学生打开网页（聊天界面）
    ↓
输入一个问题，例如：「傅里叶变换和拉普拉斯变换有什么区别？」
    ↓
助手在「教材库」里找到相关内容，组织成回答，并写上「参见 Lecture 8、第 X 页」
    ↓
（可选）助手说：「你刚问了傅里叶相关，建议接下来学 Lecture 9，或做 Homework 8」
    ↓
学生也可以选择「查看本书知识框架」，看到整门课的章节与知识点结构
```

一句话：**你提问 → 助手按教材回答并注明出处 → 必要时推荐下一步学习/练习 → 需要时还能看整本书的知识框架。**

---

## 首版做哪些，以后再做哪些？

| 首版就要有（演示用） | 以后再加 |
|----------------------|----------|
| 针对《信号与系统》的答疑 + 标明出处 | 多门课、多本教材 |
| 按「先修关系」推荐下一步学/练 | 根据你的错题/薄弱点做推荐 |
| 对《信号与系统》教材生成并展示知识框架 | 用户自己上传一本教材，系统生成那本书的框架 |
| 聊天式网页，15 分钟能演示完 | 学情小结、更复杂的诊断等 |

---

## 和「普通搜索引擎 / 通用 ChatGPT」的区别

- **普通搜索 / 通用聊天**：不一定按你这门课、这本教材来，出处也不一定靠谱。
- **我们的助手**：**只依据我们灌进去的课程材料**（首版是《信号与系统》的讲义和习题）来回答，并且**必须标出是教材里哪一章、哪一页**，方便你翻书核对，适合正经学习、备考。

---

## 小结（给组会/汇报用）

- **产品**：《信号与系统》课程助手（答疑 + 推荐 + 知识框架）。
- **核心**：按教材回答、标出处、推荐下一步、能看整本书知识结构；设计上为以后的「薄弱点分析」留了位置。
- **首版**：一门课、一本教材、聊天式网页，能在一周内做出可演示的版本。
